# Ithos

Welcome to this project's engineering memory.

The .ithos directory preserves the engineering context behind this repository.

Unlike source code, which records what changed, Ithos records why it changed.

This directory exists for both humans and AI agents.

---

# Purpose

Use Ithos to capture information that future engineers will find valuable,
including:

- Engineering decisions
- Product decisions
- Architecture changes
- Patterns
- Lessons learned
- Regressions
- Defects
- Known gaps
- Release summaries

Capture signal, not noise.

---

# AI Agents

Before writing to this directory:

- Read the project documentation.
- Understand the current project context.
- Generate concise, factual, human-readable markdown.
- Prefer updating existing knowledge over creating duplicates.
- Never fabricate information.
- Do not log every action—summarize meaningful outcomes.

Every artifact should answer one question:

> Will this help someone understand how this software became what it is today?

If not, it probably doesn't belong here.

---

# Humans

Review AI-generated artifacts.

Engineering judgment always belongs to humans.

AI assists.

Humans decide.

---

# Source of Truth

Markdown is the canonical source of truth.

Everything else—dashboards, indexes, search databases, or future cloud
services—must be derived from these files.

Your engineering memory belongs to you.
